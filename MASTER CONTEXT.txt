## WHO I AM & WHAT WE'RE BUILDING

I am building the Penguin Suite — a collection of 9 music production VST plugins and desktop apps built with AI assistance. The suite is designed for music producers, with a focus on ease of use, deep DAW integration, and AI-powered features.

All plugins output VST2 and VST3 (and AU on Mac).
VST2 is included specifically for MPC2 (Akai) compatibility — personal use only, not for redistribution due to Steinberg licensing.

The name "Penguin Suite" is kept as-is. Trademark review deferred until commercial release.

The primary producer is a techno producer. Every plugin in the suite must include techno presets alongside Boom Bap, House, Pop and D&B. Techno BPM range: 130-145. Techno references: Drumcode, Afterlife, Berghain sound.

Primary DAW: MPC2 (Akai) — keep this in mind for all compatibility decisions.
Primary machine: Windows PC, Nvidia RTX 3060, running Ollama with a local LLM (4b or 8b model — check which is running with `ollama list`).

## TECH STACK

### VST Plugins
- JUCE framework — C++, compiles to VST2, VST3, AU, Standalone from one codebase
- VST2 for MPC2 compatibility (personal use only)
- pybind11 for Python/C++ bridge where needed (PenguinSplit)
- Demucs + CUDA for GPU stem separation (PenguinSplit)
- Salamander Grand Piano samples for piano sound (PenguinMIDI VST, PenguinSynthesia)

### Desktop Apps
- Electron or Tauri — cross platform Windows/Mac
- Local LLM via Ollama — Mistral 7b or Llama 3 8b
- Freesound API — free, needs one API key (PenguinFind)
- IMSLP — public domain MIDI files for classical library (PenguinSynthesia)

### Shared Across All Plugins
- Shared config file on user machine — all plugins read: Key, Scale, BPM, Genre preset
- Shared DSP utilities — scale logic, BPM sync, MIDI helpers
- Consistent Penguin visual identity

## PLUGIN 6 — PENGUINDJ

### What it does
Fully automatic DJ VST. Select a folder, press play. Handles BPM matching, harmonic mixing via Circle of Fifths, automatic transitions. Records output directly into DAW.

### Form
VST2 + VST3

### Core Flow
1. Select folder of audio tracks
2. Auto-analyse: BPM, key, energy level per track
3. Build optimal play order via Circle of Fifths scoring
4. Mix continuously with automatic transitions
5. Press play — do nothing else

### Key Matching Priority (no shifting unless absolutely necessary)
1. Same key — mix freely
2. Circle of Fifths neighbours — compatible, no shifting
3. Relative major/minor — same notes, no shifting
4. Pitch shift — last resort only, flagged visually

### Transition Types
- Classic crossfade
- Filter sweep (high pass out, low pass in)
- Echo out / drop in
- Cut on the beat
- Spinback

### PenguinSplit Integration
- Strips drums from outgoing track during transition
- Prevents drum clash
- Stem-aware blending

### Controls
- Folder selector
- Energy curve: Chill / Balanced / Peak Time
- Transition style: Smooth / Hard / Random
- Skip button
- Lock track button

### Time Stretch Limits (avoids artefacts)
- Within 5% BPM difference → stretch freely, undetectable
- 5-15% BPM difference → stretch with caution, yellow indicator
- Beyond 15% → no stretch, use alternatives:
  - Half/double time matching (90 BPM matches 180 BPM naturally)
  - Skip to next compatible track
  - Switch to hard cut transition on downbeat
- Per-track compatibility indicator: 🟢 / 🟡 / 🔴

### Additional Features
- Stem-aware mixing — bring in bass of track B before full track, drop vocals of track A before transition
- Energy curve editor — draw custom energy shape over time (build 20min, peak 10min, cool down 5min)
- Loop points — AI detects natural loop points, loops section while next track analyses
- BPM half/double time matching — musical solution for big BPM differences
- No dead air guarantee — loop points prevent gaps between tracks

## APP 1 — PENGUINSYNTHESIA

### What it does
AI-powered piano teaching desktop app. Visual falling notes, MIDI keyboard input, strict AI teacher. Songs locked behind tiers, unlocked by completing previous tier. Classical library from IMSLP (public domain).

### Form
Desktop app — Windows and Mac
MIDI keyboard required (no on-screen keyboard — keeps it serious)
Salamander Grand Piano for playback

### AI Teacher Personality
- Strict — tough love, no fluff, short direct feedback
- Local LLM: real-time responses (timing, notes, velocity)
- Claude: deep theory explanations on demand only (saves tokens)
- Adjusts slightly per genre: classical = phrasing, producer = groove

### Example Teacher Phrases
- "Wrong note on beat 2. Again from the top."
- "85% required. You got 71%. Again."
- "Stop guessing. Learn the notes first."
- "...Not bad." — reserved for completing La Campanella only

### Demo System
- No demo by default
- Request with "show me" — teacher plays perfect MIDI rendition
- Demo penalty: threshold raises from 85% to 90%
- "Fine. Watch carefully. You won't get this grace again."

### Feedback Layers
Layer 1 — Real-time (local LLM):
- Wrong notes in red
- Timing deviation in ms
- Velocity feedback
- Rushes/drags per beat

Layer 2 — End of attempt (local LLM):
- Accuracy % per bar
- Timing/velocity consistency scores
- Weakest bar identified
- Comparison to previous attempt

Layer 3 — Deep explanation (Claude, on demand):
- Music theory in producer language
- References known songs/samples
- Finger technique
- Only when user asks

### Practice Modes
- Performance — play through
- Practice — stops at mistakes
- Loop — loops selection
- Hands separately

### Progression Rules
- 85% accuracy required to advance
- Speed locked: 60% → 80% → 100% unlocks
- Hands separately before together
- 3 perfect consecutive runs to pass
- Penalty mode: wrong note = +5 seconds on timer

### Ghost Hands
- Hidden by default
- Unlocks after 5 failed attempts in a row
- Disappears after 30 seconds
- "I'll show you once. Remember it."

### Progress Tracking
- Per-song accuracy history
- Timing/velocity consistency scores
- Notes played per session
- Streak tracking

## PLUGIN 9 — PENGUINCHOP

### What it does
Full sample chopper and pad instrument. Load or record a sample, chop it using multiple modes (all snapping to zero crossings for clean cuts), arrange slices on a flexible pad grid, play via MIDI, export to MPC2 and any DAW.

### Form
VST2 + VST3

### Getting Audio In
- Drag and drop WAV/AIFF/MP3 from disk
- Built-in file browser
- PenguinFind integration — load directly from search
- Recent files list
- Record directly — captures from DAW input or track
- Auto-trims silence from recorded audio

### Waveform Display
- Large waveform view across top
- Zoom in/out
- Slice markers as vertical coloured lines per mode
- Click between markers to preview slice
- Drag markers to adjust position
- Right click to delete marker
- Click empty space to add manual marker

### Chop Modes (all snap to nearest zero crossing)
- Zero Crossing — slices at zero amplitude points, minimum slice length control
- Transient Detection — slices at attack peaks, sensitivity + pre-roll control
- Equal Division — 4, 8, 16, 32, 64 equal slices
- Bar/Beat Division — uses suite BPM, slice per bar/beat/1/8/1/16
- Manual Tap — hit CHOP button by feel while sample plays, snaps to zero crossing
- Smart Chop (AI) — local LLM finds most musical moments, choose how many slices

### Manual Tap / CHOP Button
- Play sample, tap CHOP button by feel when you hear a good chop point
- Marker placed at tap point, auto-snapped to nearest zero crossing
- Snap range: Tight (5ms) / Normal (20ms) / Loose (50ms)
- Quantize taps — snap to nearest beat after the fact
- Undo last tap button
- Human reaction time compensation — slight pre-roll correction

### Flexible Pad Grid
- One pad per slice, up to 64 pads
- Mini waveform shown per pad
- MIDI note mapping shown per pad

### Per Pad Controls
- Volume, Pan, Pitch (±24 semitones), Fine tune (±100 cents)
- Reverse, Loop, One shot toggles
- Start trim, End trim
- Fade in, Fade out
- Mute, Solo
- Choke groups
- Round robin
- Randomize

### MIDI Mapping
- Default: pad 1 = C2, pad 2 = C#2 (MPC style)
- Customise any pad to any note
- Velocity sensitive
- Velocity to pitch option

## PLUGIN 12 — PENGUINKICK

### What it does
Kick drum designer VST. Synthesize kicks from scratch, layer with samples, or combine both. Simple 6-knob interface for instant results, advanced view for full control. Techno-focused but covers all genres.

### Form
VST2 + VST3

### Two Modes
- Simple view — 6 knobs, sounds great immediately
- Advanced view — full synthesis control, toggle between views

### Simple View — 6 Knobs
- PUNCH — low frequency impact and transient weight
- TONE — fundamental frequency and pitch envelope speed
- CLICK — attack transient character and intensity
- LENGTH — body decay and tail sustain
- RUMBLE — sub frequency weight (20-50hz), chest-moving presence
- DUCK — automatic sidechain ducking on kick hit, pump feel

### Generation Modes
- Synthesized — pure synthesis, no samples needed
- Sample + synthesis hybrid — load sample as base, layer synthesis on top
- Both modes available, switchable per preset

### Advanced View — Body Section
- Oscillator type: Sine, Triangle, custom draw
- Pitch start frequency (Hz)
- Pitch end frequency (Hz)
- Pitch curve: exponential, linear, logarithmic
- Pitch decay time (ms)
- Visual pitch curve editor — draw exact drop shape, multiple breakpoints
- Body distortion type + amount
- Body volume

### Advanced View — Click Section
- Click type: Sine, Noise, Distorted noise, Sample
- Click frequency
- Click decay (ms)
- Click filter HPF frequency
- Click volume
- Click distortion

### Advanced View — Sub Section
- Pure sine sub oscillator
- Sub frequency — locked to body or independent
- Sub decay
- Sub volume
- Sub filter

### Advanced View — Rumble Section
- Rumble frequency: 20-50hz range
- Rumble decay: 500ms to 2 seconds
- Rumble curve control
- Rumble distortion — adds harmonics for translation on small speakers
- Rumble HPF — protects speakers below 20hz
- Rumble volume

### Advanced View — Duck Section
- Duck amount (db)
- Duck attack (ms) — usually 1-5ms
- Duck release (ms) — synced to BPM (1/4, 1/8 note)
- Duck curve: exponential, linear
- Frequency selective ducking — duck low-mids, leave highs
- Sidechain output — triggers PenguinFX sidechain compressor automatically

### Advanced View — Noise/Dirt Section
- Background noise layer: vinyl crackle, tape hiss, room tone
- Decay and volume
- Adds analogue character

### Advanced View — Amplitude Envelope
- Attack, Hold, Decay
- Visual envelope display

### Advanced View — Distortion/Processing
- Pre and post distortion
- Types: soft clip, hard clip, tape, tube, foldback
- Oversampling toggle

### Advanced View — Layering
- Up to 3 independent layers
- Each layer: synthesis or sample
- Per layer: volume, pan, pitch offset

### Advanced View — Extras
- Pitch modulation — subtle analogue wobble, rate + depth
- Transient shaper — sharpen/soften attack post-synthesis
- Built-in compression — one-knob glue
- Sidechain output — dedicated signal for external sidechain routing

## NOTES & DECISIONS LOG

- VST2 included for MPC2 — Akai has been slow to add VST3 support
- Key shifting in PenguinDJ avoided unless absolutely no alternative
- PenguinVoice AI pitch correction is OFF by default — never automatic
- PenguinFind searches toggled independently — local search off by default to avoid constant rescanning

- Monorepo: one Claude Project = whole suite stays on same page
- Each package gets own Docker when development starts — not upfront
